TAU-bench evaluates Agents in real-world environments

TAU-bench evaluates Agents in real-world environments and showed poor reliability. It tests if an agent can reliably engage in a dynamic, multi-turn conversation with a user to figure out what needs to be done.

T-bench:

1Ô∏è‚É£ Agent interacts with an simulated user to understand needs & gather info over multiple turns.

2Ô∏è‚É£ Agent utilizes domain-specific API tools (e.g., book flight, return item)

3Ô∏è‚É£ Agent must adhere to a provided policy document containing domain-specific rules and restrictions.

4Ô∏è‚É£ Success is measured by comparing the final database state

5Ô∏è‚É£ Uses the pass^k metric to evaluate reliability on the same task over multiple (k) trials.

Insights

  • üìâ At release agents succeeding less than 50% of the time.

  • üßÆ Includes 10-15 tools across 50-115 different tasks for a retail and airline domain.

  • ü§∑ Agents are inconsistent, failing on tasks they previously succeeded on, low pass^k scores (pass^8 < 25%).

  • üìä Tasks requiring 4+ database writes have only ~20% success rate vs. ~75% for single-action tasks.

  • üöß Failures stem from poor reasoning about database states, misunderstanding or ignoring rules (policy adherence), or mishandling complex multi-step requests.

  • üìù Removing domain guidelines drops performance by 22% in complex domains.
